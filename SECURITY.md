# Security policy

Please report suspected vulnerabilities privately to the repository owner before opening a public issue. Include the affected version, target framework, reproduction steps, and any relevant theme-pack or signature input without sharing private signing keys.

Do not commit private keys or credentials. Theme-pack signing keys are release secrets and must remain outside the repository.

Supported versions:

- `0.3.x`: current public-release line
- older versions: best effort only; upgrade before reporting issues that reproduce on `0.3.x`
