# CopperSkin v0.3.0.0 Scope

v0.3.0.0 is the first official public release target for the upgraded CopperSkin library. The release combines a reusable WPF graphics surface with a practical Designer authoring workflow and a measured expansion of standard-control coverage.

## Graphics editor minimum

- Create transparent icon documents at 16, 20, 24, 32, 48, and 64 pixels.
- Create arbitrary basic-paint canvases with a background and ordered layers.
- Draw and render rectangles, ellipses, lines, freehand strokes, paths, and text.
- Select existing elements through document-space hit testing.
- Undo and redo edits through serialized Core document snapshots.
- Save and reopen .cgraphic/JSON documents.
- Render the same Core document through CopperSkin.Wpf GraphicCanvas or CopperIcon.
- Export SVG/XAML/PNG in the completed release workflow.

The first editor does not attempt to be a full raster illustration package. Filters, gradients, animation, collaboration, and a high-performance raster brush engine are deferred.

## WPF coverage minimum

The v0.3 coverage wave adds safe default styling for BulletDecorator, Image, InkPresenter, and MediaElement and improves behavior coverage for common popup/data/document controls in follow-on slices. External media hosting remains application-owned and is not represented as full custom control replacement.

Every control claimed as supported needs evidence for resource loading, focus/keyboard behavior, validation states, disabled/read-only states, high-DPI rendering, RTL layout, and theme switching. Unsupported or partial surfaces stay marked as such in CONTROL_COVERAGE.md.

## Compatibility

- Existing Core and WPF target frameworks remain supported.
- Existing theme packs without graphics remain readable and do not gain an empty graphics property when written.
- Graphics schema versioning is independent from the assembly version.
- Existing scoped-theme activation and corrected horizontal-scroll behavior remain unchanged.
- Assembly, file, informational, theme-pack, and compatibility versions are `0.3.0.0`; NuGet normalizes the package identity to `0.3.0`.
